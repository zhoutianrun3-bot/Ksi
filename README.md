# Ksi

Ksi 是一个零加价、无默认捆绑的机票搜索与旅程钱包原型。它把搜索、价格解释、航变提示和登机牌管理留在 Ksi，把收款、出票、退款及拒付交给具备资质的出票伙伴。

## 当前版本

这是可运行的产品首版，不是假装已经接入航司的静态效果图。演示数据通过独立数据层提供，真实供应商可以依照 `src/services/contracts.ts` 接入，而不必重写界面。

已实现：

- 无限添加和删除串联行程；
- 预设 0 / 20 / 30kg 托运行李；
- 全服务、LCC 或混合筛选；
- 仅以颜色表达日期价格区间，并保留无障碍语义；
- 点击每个航班号查看该航班、该日期的历史价格；
- 最优支付币种及原始报价币种并列显示；
- 低价、时刻观察和已航变的独立视觉状态；
- 固定位置展示航变解决方案；
- Apple Liquid Glass 与 Pixel Material 3 两套设备主题；
- Apple Pay、Google Pay、云闪付的出票伙伴支付交接界面；
- Apple / Google 登录，以及中国大陆临时支付宝认证入口；
- Apple Wallet 风格的分层旅程与登机牌主页；
- 响应式、可安装 PWA。

## 不可伪装的边界

以下功能必须在签约与合规完成后才能启用，当前界面只演示流程：

- 航司 NDC/GDS/LCC 实时库存与重新核价；
- 出票伙伴真实订单、支付链接、PNR 和电子客票；
- Apple、Google、支付宝的生产身份凭证；
- 自然人级唯一性和成年验证；
- 航变、退款、改签 webhook；
- 多币种支付与原币退款。

Ksi 不应保存卡号、支付钱包令牌、身份证件、人脸、支付宝账单或芝麻信用数据。

## 项目结构

```text
src/
  App.tsx                 产品交互与主要界面
  data.ts                 可替换的演示航班与行程数据
  types.ts                前端领域模型
  lib/pricing.ts          中立筛选与价格规则
  services/contracts.ts   搜索、出票、身份与航变接入合同
  styles.css              Apple / Pixel 自适应视觉系统
docs/
  PRODUCT.md              产品、数据与治理原则
```

## 运营原则

Ksi 服务费为 0；结果从第一屏显示全部应付金额；返佣不影响排序；所有结余只用于合理薪酬、运营储备、基础设施和降低用户成本。出票伙伴始终作为 Merchant of Record。
