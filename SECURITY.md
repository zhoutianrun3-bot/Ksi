# Security Policy

Ksi 涉及旅行身份、订单和支付交接，因此数据最小化优先于事后补救。

## 支持范围

当前 `main` 分支上的最新版本接受安全修复。原型尚未连接真实航司、身份或支付生产环境。

## 报告问题

在正式安全邮箱公布前，请不要通过公开 Issue 披露以下内容：

- 可用于冒用账户的身份或登录问题；
- 订单、PNR、电子客票或乘机人信息泄露；
- 支付跳转、收款方显示或退款币种被篡改；
- 航变 webhook 重放或伪造；
- 可绕过“一名成年人一个有效账户”的缺陷。

仓库所有者应在公开仓库前配置 GitHub Private Vulnerability Reporting，并在本文件补充专用安全联系方式。

## 永不进入 Ksi 的数据

- 完整银行卡号、CVV 和持久支付钱包令牌；
- 身份证、护照原件图片及人脸模板；
- 支付宝账单、余额、联系人或芝麻信用；
- 第三方 OAuth 客户端密钥；
- 可从日志重建的完整票务或身份凭证。

发现上述内容进入日志、测试数据或版本历史时，应按凭证泄露处理，而不是仅删除当前文件。
