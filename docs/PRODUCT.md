# Ksi 产品与系统基线

## 一句话定义

Ksi 是成本回收型的中立航班搜索与旅程管理层：不加价、不捆绑、不保存支付卡，由出票伙伴直接收款与出票。

## 体验原则

1. 搜索不要求登录。
2. 每次结果都显示最终应付价，而不是诱导点击的基础价。
3. 价格行情主要用连续色阶表达；金额本身仍保留，色觉辅助信息写入无障碍标签。
4. 每个航班号都提供“航班号 + 具体出发日”的历史价格，不把不同日期混成一条曲线。
5. 航变卡片同时显示状态和解决方案，解决方案不藏在帮助中心。
6. 行李额是搜索条件，不是结账时的加购陷阱。
7. 平台主题跟随设备：Apple 使用玻璃、景深和分层；Pixel 使用 Material 3 色调面与大圆角。

## 身份

Ksi 内部只有一个 `ksi_person_id`。Apple、Google 与中国大陆过渡期的支付宝只是可替换登录凭证。

- 邮箱不是主键；
- 支付宝账户 ID 不是自然人唯一凭证；
- 一人一号必须依赖合规认证方返回的自然人级不透明 `person_key`；
- Ksi 不自己对身份证号做哈希，因为低熵证件号仍可能被反推或撞库；
- 一个账户可以绑定多个身份入口；
- 删除、解绑与账号恢复必须在应用内完成；
- 支付宝入口退役前至少提供一段迁移期，行程和订单不随登录入口消失。

## 付款与出票

```text
Ksi 搜索并标准化报价
  → 出票伙伴重新核价并创建订单
  → 用户在伙伴的 Apple Pay / Google Pay / 云闪付界面付款
  → 伙伴出票并回传 PNR / 电子客票号
  → Ksi 更新旅程钱包
  → 改签退款由 Ksi 发起、伙伴执行
```

钱包或银行卡账单应显示出票伙伴名称。Ksi 只保存伙伴订单号、金额、币种、状态和必要的票务引用，不保存卡号与可再次扣款的支付凭证。

## 货币路由

只比较真实可支付、符合销售地规则的报价：

1. 获取每个合法销售点与币种的最终含税价；
2. 使用带时间戳的中间市场汇率统一比较；
3. 加入出票伙伴明确披露的费用；
4. 假设用户使用免外币转换费的国际卡；
5. 禁止动态货币转换（DCC）；
6. 在付款前显示原始交易与退款币种；
7. 不伪造居住国、账单地址或销售地。

## 航变

航变 webhook 必须是幂等事件。每个事件至少包含原时刻、新时刻、出票规则和伙伴可执行的补救措施。Ksi 只展示伙伴确认可履行的选项，不自行承诺赔偿。

## 治理

- Ksi 服务费固定为 0；
- 返佣、接口补贴和赞助不得参与排序；
- 年度报告披露运营成本、储备范围、供应商覆盖率与主要中断；
- 公司章程禁止利润分配，并为使命条款设置独立否决权；
- 清算后的剩余资产不得分配给股东。

## 上线门槛

- 至少两家出票伙伴或一家伙伴加航司直连，避免单点供应；
- 重新核价成功率、出票时延、退款时延和 webhook 丢失率有可观测指标；
- 全链路不记录身份令牌、证件内容或支付敏感字段；
- 通过移动端无障碍、弱网、离线钱包和大字体测试；
- 明确每个销售地区的旅行代理、消费者保护、数据和支付责任。
