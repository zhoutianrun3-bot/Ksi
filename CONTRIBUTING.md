# Contributing to Ksi

感谢你愿意帮助 Ksi。项目的首要目标不是提高成交额，而是让用户更快、更清楚地理解机票的最终价格、行李条件和航变风险。

## 开始开发

需要 Node.js 22 或更新的长期支持版本。

```bash
npm ci
npm run dev
```

提交修改前运行：

```bash
npm test
npm run build
```

## 设计约束

- 不加入默认勾选、倒计时施压或隐藏费用；
- 不以佣金、赞助或合作关系改变自然排序；
- 不保存银行卡号、钱包支付令牌、证件图片或人脸；
- 价格色彩必须同时保留无障碍名称，不能只让色觉正常用户理解；
- 航变方案必须来自可实际履行的出票伙伴规则；
- 第三方供应商实现应放在独立适配层，不能污染 Ksi 的统一领域模型；
- 新依赖应说明必要性，并避免显著增加首屏体积。

## Pull Request

PR 应说明：

- 修改了什么；
- 对用户的影响；
- 是否接触身份、支付或票务数据；
- 使用了哪些测试；
- 是否改变零加价、中立排序或隐私边界。

涉及真实航司、出票、身份或付款接口的示例不得包含生产密钥或真实个人数据。
